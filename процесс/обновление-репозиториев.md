# Обновление репозиториев перед проходкой

Пользователь может сказать агенту: обнови все репозитории и сделай новую проходку.

## Правило

Перед обновлением смотри `инвентаризация/репозитории.md` и карты репозиториев. Там указаны основная ветка, последняя просмотренная версия и команда обновления.

Если есть `.local/repository-tracking.md`, используй его как локальный источник политики обновления:

- постоянно отслеживаемые можно подтягивать полуавтоматически перед проходками;
- отслеживаемые подтягиваются только по явному контексту проходки или просьбе пользователя;
- неотслеживаемые не подтягиваются и не читаются без явной просьбы пользователя.

## Пример команды

```text
git -C ../имя-репозитория pull origin master
```

Если основная ветка `main`, используй `main`. Если ветка неизвестна, добавь вопрос в `решения/требует-уточнения.md`.

## После pull

- Обнови `Последняя просмотренная версия`.
- Обнови `Дата pull`.
- Обнови карту репозитория.
- Запиши проходку в `задачи/журнал-исследований.md`.
- Проверь, какие утверждения могли устареть.

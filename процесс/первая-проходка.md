# Первая проходка

Цель: собрать полезную карту и список вопросов, не притворяясь, что система уже понята.

## Шаги

0. Проверить `.local/repository-tracking.md`.
   - Если файла нет, создать его по шаблону `шаблоны/локальная-классификация-репозиториев.md`.
   - Для создания пройтись по соседним git-репозиториям и предположить категорию: постоянно отслеживаемые, отслеживаемые, неотслеживаемые.
   - После создания остановиться и написать пользователю: `Открой .local/repository-tracking.md, вручную проверь классификацию репозиториев и подтверди, что можно продолжать первую проходку.`
   - Не продолжать первую проходку без подтверждения.
1. Прочитать `KB_INDEX.md` и `AGENTS.md`.
2. Прочитать подтвержденный `.local/repository-tracking.md`.
3. Заполнить `инвентаризация/репозитории.md` только подтвержденными отслеживаемыми репозиториями.
4. Создать карты репозиториев из `шаблоны/карта-репозитория.md`.
5. Читать сначала маленькие источники: README, инструкции агента, решения, манифесты, точки входа, тесты, конфиги.
6. Найти первые бизнесовые понятия и завести страницы в `бизнес/` или `области-знаний/`.
7. Обновить `архитектура/зависимости-репозиториев.md` только фактами с источниками.
8. Все неизвестное добавить в `решения/требует-уточнения.md`.
9. Обновить `глоссарий.md` и `ссылки.md`.
10. Записать итог в `задачи/журнал-исследований.md`.

## Ограничения

- Не читать весь код без цели.
- Не делать выводы по названию репозитория.
- Не включать неотслеживаемые репозитории в публичную базу знаний без явного решения пользователя.
- Не копировать длинные документы.
- Не смешивать бизнесовые правила и техническую реализацию без ссылок между ними.

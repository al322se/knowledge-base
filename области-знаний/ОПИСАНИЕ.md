# Области знаний

Область знаний описывает домен, который может проходить через несколько репозиториев и источников.

Область может быть:

- технической: авторизация, интеграции, миграции, деплой, наблюдаемость;
- бизнесовой: заявки, тарифы, договоры, статусы, пользовательские роли;
- смешанной: биллинг, онбординг, каталог, заказы, модерация.

## Когда создавать область

- Один бизнесовый процесс реализован в нескольких репах.
- Один термин встречается в коде, документах и разговорах.
- Изменение требует проверки нескольких систем.
- Нужно связать бизнесовое правило с технической реализацией.

Используй шаблон [область знаний](../шаблоны/область-знаний.md).

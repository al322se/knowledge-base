# Repository Cards

One file per source repository. Create initial cards from `templates/repo-card.md`.

Keep cards concise. Move deep implementation notes into linked system or architecture pages.

## Naming

Use the repository folder name unless there is a strong reason to normalize it.

Examples:

- `repos/api-gateway.md`
- `repos/auth-service.md`
- `repos/frontend-app.md`

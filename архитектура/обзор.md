# Обзор архитектуры

Эта страница должна оставаться верхнеуровневой.

## Что здесь описывать

- крупные части системы;
- границы между репозиториями;
- основные потоки данных;
- владение бизнесовыми сущностями;
- внешние интеграции;
- ссылки на подробные страницы.

## Что не делать

- Не дублировать карты репозиториев.
- Не описывать бизнесовые правила без ссылки на бизнесовую страницу.
- Не утверждать архитектуру без источников.
